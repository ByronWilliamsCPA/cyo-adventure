## Standards conformance summary (vs testing.md v2.0)

MUST-level failures:

1. §1.2 pytest config: missing `filterwarnings = ["error"]`, `xfail_strict = true`,
   `--import-mode=importlib`; 4 markers registered in conftest instead of pyproject.
2. §1.3 dependencies: `pytest-mock` absent (while tests/CLAUDE.md instructs its use).
3. §4.2 mock spec discipline: ~1% compliance (3 of ~267 mock sites set `spec=`;
   zero `autospec`).
4. §2.3 assertions: 271/368 `pytest.raises` without `match=`; 33 float equality
   comparisons without `pytest.approx`; 19 assert-free "does not raise" tests;
   1 literal `assert True` (tests/test_example.py:168).
5. §6.2/§14.7 critical modules: two below threshold (table above); no parametrized
   role x endpoint x method authorization matrix (per-module handwritten 403 tests
   only).
6. §11.2 AI tiers: no Tier 3 behavioral suite, `llm_eval` and `ai_security` markers
   unregistered and unused.
7. §14.8: no auth-event-logging tests and no secrets-never-logged (caplog) tests.
8. §6.1 (frontend): Vitest defines no coverage thresholds; only Codecov's 70% line
   target on the `frontend` flag enforces anything.

PARTIAL / SHOULD-level:

- §4.3 boundary mocking: strong seams (testcontainers Postgres, MockProvider,
  httpx.MockTransport, real Redis testcontainer) but internal-stage patching in
  moderation-pipeline tests, private-function patching in worker tests, two direct
  third-party patches (Anthropic SDK method, genai.Client), and 14 duplicated
  `_FakeSession` definitions across unit files.
- §3.2 conftest: dead template scaffolding (fixtures_dir/benchmarks_dir pointing at
  nonexistent data/), domain helper in root conftest, engine fixture teardown not
  try/finally.
- §7.4: pytest-asyncio strict mode with explicit markers (documented local
  convention; acceptable deviation).
- Frontend: fireEvent dominates userEvent 158:39 on older form tests; 21% testid
  queries; 3 CSS-class querySelector assertions (FlagBadge); raw 300ms setTimeout
  sleeps in 4 CI e2e naive-user specs; no MSW (no composed page+API+interceptor
  path test).

Strengths verified: two-family seed fixture exercises horizontal family isolation
across 10+ API modules; error-path coverage on all sampled API modules; zero
time.sleep in backend tests; zero direct os.environ mutation; prior findings
"replayQueue unwired" and "RQ/Redis faked everywhere" both resolved.

## Remediation scope (this branch)

Batch A — backend coverage floors:
A1 covers module (worker/service/prompt) to floor; A2 moderation/report.py and
classifiers.py to 90/80 including Finding and branch gaps; A3 remaining function
gaps across api/, middleware/security.py, generation/, validator/,
story_requests/.

Batch B — standards conformance:
B1 pytest config (filterwarnings=error with warning fixes, xfail_strict,
import-mode, marker consolidation, pytest-mock dep, conftest cleanup,
llm_eval/ai_security marker registration); B2 assertion mechanics (assert-free
tests, pytest.approx, assert True, match= adoption prioritized on project-owned
exceptions); B3 mock hygiene (spec= adoption on worst files, de-mock
moderation-pipeline stage stubs to run real stages with MockProvider, consolidate
_FakeSession into shared spec'd fake); B5 new authz role x endpoint matrix test
plus auth-event logging and secrets-not-logged tests.

Batch C — frontend:
C1 Vitest thresholds (70% floors), tests for the below-floor files and the 22
uncovered functions, coverage exclusions for main.tsx and type-only modules with
rationale, replace raw sleeps in CI e2e specs with locator waits.
