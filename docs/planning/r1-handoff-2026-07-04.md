## Conventions and lessons the new session must carry

- Signed commits (`git commit -S`), Conventional Commits, NO em-dash anywhere, stage only named
  files, pushes and merges user-gated, worktrees at .worktrees/<slug>.
- SDD loop per docs/planning/r1-gap-closure-plan.md: fresh implementer per task (haiku for
  transcription, sonnet standard, opus for plans/final reviews), per-task review with spec +
  quality verdicts, fix waves for Critical/Important, Minors to the ledger, final whole-branch
  opus review, ledger at `<worktree>/.superpowers/sdd/progress.md` (git-ignored: copy plans into
  new worktrees manually; ignored files do not travel with branches).
- Guardian redaction discipline: only category/verdict/message cross to guardians; kid surfaces
  get even less (id/status). A TypeScript cast is NOT redaction: strip at runtime with .map and
  pin with an Object.keys regression test (K1 lesson).
- Double-submit protection is layered: synchronous in-flight guard set before the first await in
  the UI, `SELECT ... FOR UPDATE` row lock before the pending-status guard in the backend
  (api/reading.py:208 pattern).
- Plan snippets go stale: two referenced a nonexistent Button variant ("secondary"; real union is
  primary/ghost/danger), one had a stale e2e auth mock. The real codebase always wins; document
  deviations in the task report.
- Adversarial fixture design: a cross-family authz test must use fixtures where the wrong and
  right values actually diverge (B7 lesson: same-family admin fixture made the stamping
  assertion a no-op oracle).
- Concurrent sessions are active in this repo: re-fetch origin heads before touching any branch,
  never assume a worktree or tip is as you left it, and check `gh pr view` before pr-fix work.
- e2e auth patterns differ by route tree: guardian routes need the Supabase session-seeding
  pattern (assignments.spec.ts); kid library routes use plain auth_token localStorage
  (library.spec.ts). Wire paths are /api/v1/... (axios baseURL '/api').
